# 基于线性代数剪枝的图中套利回路搜索研究报告

## 摘要

本项目研究的问题是：在一个由多种货币或资产构成的兑换网络中，如何快速寻找高收益的套利回路。我们将货币视为图的顶点，将兑换关系视为有向边，边权为汇率。若一条闭合路径上的汇率乘积大于 1，则该路径构成套利回路。传统方法需要在全图中搜索大量简单回路，随着节点数增加会迅速变得不可计算。为解决这一问题，本项目提出两种线性代数剪枝方法：**Lap 残差剪枝算法**与**SVD 秩一近似剪枝算法**。两种方法都先从汇率矩阵中提取“热点节点”，再在热点节点诱导子图中搜索最优套利回路。这样既利用了线性代数对全局结构的刻画能力，又保留了图论中“回路”这一核心对象。实验表明，在小规模图中，诱导子图方法能显著缩短搜索时间；在大规模图中，当全图 DFS 已经需要天文级时间时，剪枝算法仍能在毫秒级给出较高收益的回路。项目还利用概率论估算随机噪声背景下最佳简单回路的理论收益，为大规模实验提供了理论参照。

**关键词**：图论；套利回路；汇率矩阵；SVD；Lap 残差；诱导子图；剪枝算法；概率论估计

---

## 一、问题背景与研究目标

套利是金融市场中常见的问题。假设存在若干种货币或资产，如果从资产 (A) 出发，依次兑换为 (B,C,\dots)，最后回到 (A)，并且最终获得的 (A) 多于最初投入的 (A)，那么这条兑换路径就是套利路径。

例如：

[
A\to B\to C\to A
]

若有

[
R_{AB}R_{BC}R_{CA}>1
]

则该路径可以产生套利收益。

从图论角度看，这一问题可以转化为：**在一个带权有向图中寻找收益最大的有向简单回路**。其中，“简单回路”指除起点和终点相同外，中间不重复经过同一顶点的回路。

本项目的基本目标不是单纯判断“是否存在套利”，而是进一步寻找**收益尽可能高的套利回路**。这比普通负权环检测更难，因为 Bellman-Ford 或 SPFA 可以判断负环存在，但不一定直接给出全局最优收益回路。因此，实验中我们使用 DFS 枚举简单回路作为小规模图上的精确基准，并在大规模图中通过剪枝缩小搜索空间。代码中的 `calc_profit` 函数按路径逐边相乘并计算收益，DFS 则用于搜索最优简单回路。

## 二、图论建模与基本假设

设市场中共有 (N) 种货币或资产，记为：

[
V={1,2,\dots,N}
]

任意两种资产之间的兑换关系构成有向边：

[
(i,j)\in E
]

汇率矩阵定义为：

[
R=(R_{ij})_{N\times N}
]

其中 (R_{ij}>0) 表示 1 单位资产 (i) 可以兑换多少单位资产 (j)。

于是，市场可以建模为一个带权有向图：

[
G=(V,E,R)
]

对于一条回路：

[
C=i_1\to i_2\to \cdots \to i_L\to i_1
]

其总收益率为：

[
P(C)=R_{i_1i_2}R_{i_2i_3}\cdots R_{i_Li_1}-1
]

若 (P(C)>0)，则该回路为套利回路。本项目的目标是寻找：

[
C^*=\arg\max_C P(C)
]

即收益最高的简单套利回路。

本项目采用以下基本假设：

第一，市场在某一瞬间可以用固定汇率矩阵表示，不考虑汇率在搜索过程中的继续变化。

第二，初步模型中不考虑手续费、滑点和交易容量限制。

第三，实验主要针对完全或较稠密的有向兑换图。若图本身极度稀疏，则直接使用 SPFA 或 Bellman-Ford 可能更合适。

第四，实验数据采用模拟汇率矩阵生成。代码中先随机生成每种资产的基准价格，再按

[
R_{ij}=\frac{p_j}{p_i}(1+\epsilon_{ij})
]

生成带噪声汇率，其中噪声由 `noise_level` 控制；当 `trap=True` 时，会人为注入一条由 4 个节点构成的强套利结构，前三条边乘以 (1.04)，最后一条边乘以 (0.98)。

## 三、全图搜索的困难

若不剪枝，完全有向图中边数约为：

[
E=N(N-1)\approx N^2
]

如果只是检测是否存在套利，可以将边权取为：

[
w_{ij}=-\ln R_{ij}
]

则一条回路的收益乘积大于 1 等价于：

[
\sum w_{ij}<0
]

也就是寻找负权环。此时 Bellman-Ford 的复杂度为：

[
O(VE)=O(N^3)
]

但本项目更关注“最佳简单套利回路”，因此实验中采用 DFS 枚举简单环作为小规模图的基准。简单回路数量随 (N) 增长近似呈阶乘级爆炸。对于长度为 (L) 的简单有向回路，其数量约为：

[
\frac{N(N-1)\cdots(N-L+1)}{L}
]

当 (L) 可以从 2 增加到 (N) 时，总搜索空间极大。

实验中，(N=10) 的完全图全局 DFS 平均耗时已经约为 1380.69 ms；而当 (N) 增加到 50、100、200 时，全图 DFS 的估算耗时分别达到约 (10^{50}) 年、约 (10^{144}) 年，甚至接近无限时间。

因此，本项目的关键问题变成：

> 如何在不完全破坏套利环结构的前提下，提前缩小搜索空间？

---

## 四、总体算法思想：先用线性代数找热点，再用图论搜索回路

本项目最终形成的算法框架如下：

1. 输入完整汇率矩阵 (R)；
2. 用 Lap 残差算法或 SVD 秩一近似算法计算每条边的“异常程度”；
3. 根据异常边提取一组热点节点；
4. 在热点节点之间保留所有原始边，形成热点节点诱导子图；
5. 在诱导子图中运行 DFS，寻找最大收益简单回路；
6. 将剪枝结果与全图 DFS 或概率论理论基准比较。

这一思路的核心不是简单“删边”，而是“选节点”。

如果直接保留残差最大的若干条边，容易破坏回路。例如一个套利环有 5 条边，只保留其中 3 条，环就消失了。为避免这一问题，我们采用**热点节点诱导子图策略**：先通过高残差边找出可疑节点，再保留这些节点之间的全部兑换边。代码中 `_extract_hot_induced_subgraph` 先按残差从大到小排序边，再不断加入边的两个端点，直到热点节点数达到上限，最后保留这些热点节点之间的所有原始边。

因此，本项目的剪枝逻辑可以概括为：

[
\text{残差矩阵负责定位热点区域，诱导子图负责保留回路结构。}
]

## 五、Lap 残差剪枝算法

### 5.1 算法动机

Lap 算法的核心是对汇率矩阵取对数，将乘法关系转化为加法关系。定义：

[
M_{ij}=\ln R_{ij}
]

如果市场完全无套利，并且存在某种基准价格 (p_i)，则：

[
R_{ij}=\frac{p_j}{p_i}
]

所以：

[
M_{ij}=\ln R_{ij}=\ln p_j-\ln p_i
]

令：

[
a_i=\ln p_i
]

则：

[
M_{ij}=a_j-a_i
]

这说明，理想情况下，对数汇率矩阵可以由每个点的“势值差”解释。换言之，边 (i\to j) 的权重主要由终点 (j) 和起点 (i) 的基准差决定。

### 5.2 中间推导

代码中使用列均值估计每个节点的势值：

[
y_j=\frac{1}{N}\sum_i M_{ij}
]

在理想无套利情况下：

[
y_j=\frac{1}{N}\sum_i(a_j-a_i)
]

由于 (a_j) 与求和下标 (i) 无关：

[
y_j=a_j-\frac{1}{N}\sum_i a_i
]

于是：

[
y_j-y_i
=======

## \left(a_j-\frac{1}{N}\sum_k a_k\right)

# \left(a_i-\frac{1}{N}\sum_k a_k\right)

a_j-a_i
]

这正好等于理想对数汇率：

[
M_{ij}=a_j-a_i
]

因此，若市场完全无套利，应该有：

[
M_{ij}\approx y_j-y_i
]

由此定义 Lap 残差：

[
\Delta M_{ij}=M_{ij}-(y_j-y_i)
]

若 (\Delta M_{ij}) 较大，说明边 (i\to j) 的实际兑换率高于由整体市场结构推测出的理论值，这条边更可能参与套利。

代码中正是这样实现的：先计算 (M_{ij}=\ln R_{ij})，再令 `y = np.mean(M, axis=0)`，最后用 `Delta_M[i,j] = M[i,j] - (y[j]-y[i])` 作为热点节点提取依据。

### 5.3 算法步骤

Lap 剪枝算法可写为：

1. 输入汇率矩阵 (R)；
2. 计算对数矩阵：

[
M_{ij}=\ln R_{ij}
]

3. 计算列均值向量：

[
y_j=\frac{1}{N}\sum_i M_{ij}
]

4. 计算残差矩阵：

[
\Delta M_{ij}=M_{ij}-(y_j-y_i)
]

5. 按 (\Delta M_{ij}) 从大到小排序；
6. 取残差较大边的端点作为热点节点；
7. 形成热点节点诱导子图；
8. 在诱导子图中搜索最佳简单套利回路。

### 5.4 算法特点

Lap 算法的优点是推导较自然，与图论中的边权、路径、回路关系密切。它本质上是先估计图上“正常”的整体兑换结构，再寻找偏离该结构的局部异常。

Lap 算法的短处是，它依赖对数残差的局部排序。如果某个套利环中每条边的残差都不大，但累计起来收益较高，那么较浅剪枝可能漏掉部分节点。因此，Lap-6Node 虽然极快，但有时收益保留比例不如 Lap-8Node。

## 六、SVD 秩一近似剪枝算法

### 6.1 算法动机

SVD 算法从另一个角度理解汇率矩阵。若市场无套利，则：

[
R_{ij}=\frac{p_j}{p_i}
]

于是：

[
R=
\begin{pmatrix}
1/p_1\
1/p_2\
\vdots\
1/p_N
\end{pmatrix}
\begin{pmatrix}
p_1&p_2&\cdots&p_N
\end{pmatrix}
]

这说明理想汇率矩阵是一个秩一矩阵。

因此，可以用 SVD 的第一主成分重构市场的主要结构：

[
\hat R=\sigma_1u_1v_1^T
]

残差：

[
\Delta R=R-\hat R
]

则表示真实汇率对主要秩一结构的偏离。

### 6.2 实际实现

在代码中，SVD 版本并不是直接对原始 (R) 做分解，而是先做若干次行列归一化，减少不同货币尺度差异带来的影响。随后使用：

[
\hat R=\sigma_1u_1v_1^T
]

构造秩一近似，并计算：

[
\Delta R=R_{\text{bal}}-\hat R
]

最后同样使用热点节点诱导子图策略。代码中使用 `np.linalg.svd(R_bal)` 进行 SVD，并只取第一奇异值与第一组奇异向量构造 `R_hat`。

### 6.3 算法特点

SVD 算法的优点是直观：市场的主要结构由第一主成分表示，其余部分代表异常。它更能体现“用线性代数解释图中异常结构”的思想。

但 SVD 算法也有局限。第一，它主要最小化整体平方误差，可能把局部异常平滑到多个位置。第二，SVD 对矩阵尺度较敏感，所以需要归一化。第三，在某些包含“过桥边”或“局部微亏边”的套利结构中，SVD 可能不如 Lap 算法稳定。

因此，本项目没有把 SVD 作为唯一算法，而是将其作为与 Lap 算法并行的第二个剪枝版本，用实验比较二者长处和短处。

## 七、热点节点诱导子图策略

本项目的重要改进是使用“热点节点诱导子图”，而不是简单保留高残差边。

具体做法是：

1. 对所有有向边 ((i,j)) 计算残差得分；
2. 按得分从大到小排序；
3. 从最高得分边开始，不断把边的两个端点加入热点节点集合；
4. 当热点节点数达到设定上限 (m) 时停止；
5. 保留这 (m) 个节点之间的所有原始边，得到诱导子图。

若 (m=8)，诱导子图最多包含：

[
8\times 7=56
]

条有向边。这远小于原图的 (N(N-1)) 条边。

这种策略的关键价值是：

* 直接删边可能破坏套利环；
* 选热点节点再保留节点间全部边，可以尽量保留环结构；
* 搜索复杂度由原图规模 (N) 转化为热点规模 (m)；
* 当 (m) 固定为 6 或 8 时，后续 DFS 几乎变成常数级开销。

实验代码中，Lap-6Node、Lap-8Node、SVD-8Node 都采用这一诱导子图策略。

## 八、后续搜索算法

剪枝之后，算法需要在诱导子图中寻找收益最高的简单回路。

本项目实验采用 DFS 搜索。其原因是：我们的目标不是只判断是否存在套利，而是比较不同剪枝方法找到的“最大收益”。DFS 可以枚举诱导子图中的简单回路，并计算每条回路的实际收益：

[
P(C)=\prod_{(i,j)\in C}R_{ij}-1
]

对于小规模全图，DFS 作为精确基准；对于大规模图，DFS 只在 6 个或 8 个热点节点构成的诱导子图中运行。

从理论上说，如果研究目标只是判断是否存在套利，也可以使用 Bellman-Ford 或 SPFA。将边权设为：

[
w_{ij}=-\ln R_{ij}
]

则套利回路等价于负权环。但本项目更强调“高收益回路搜索”，所以 DFS 更适合作为实验中的统一比较工具。

## 九、复杂度分析

### 9.1 全图 DFS 复杂度

完全图中的简单回路数量随 (N) 增长接近阶乘级。全图 DFS 在 (N=10) 时已经需要约 1.3 秒；再往上增长会迅速不可计算。

实验中通过 (N=10) 的实际耗时进行外推，得到 (N=50)、(N=100)、(N=200) 的全图 DFS 估算耗时分别达到天文数量级。

### 9.2 Lap 剪枝复杂度

Lap 算法需要：

1. 计算对数矩阵：(O(N^2))；
2. 计算列均值：(O(N^2))；
3. 计算残差矩阵：(O(N^2))；
4. 对边残差排序：若全排序为 (O(N^2\log N^2))，若只选前若干条可近似为 (O(N^2))；
5. 在 (m) 个热点节点诱导子图中 DFS。

若 (m) 固定为 6 或 8，则 DFS 部分的开销不再随原图 (N) 增长而显著增长。因此整体主要由矩阵扫描和排序决定，约为：

[
O(N^2\log N)
]

或在优化选取时近似：

[
O(N^2)
]

### 9.3 SVD 剪枝复杂度

完整 SVD 的复杂度一般为：

[
O(N^3)
]

但本项目只使用秩一近似。理论上若采用幂迭代，只需计算最大奇异值及对应向量，可以降至：

[
O(dN^2)
]

其中 (d) 是迭代次数。当前代码中使用 `np.linalg.svd`，在实验规模内可行；若扩展到更大规模，可改为幂迭代或随机 SVD。

### 9.4 总体比较

| 方法                | 主要思想       | 复杂度特点                   | 优点               | 风险                 |
| ------------------- | -------------- | ---------------------------- | ------------------ | -------------------- |
| 全图 DFS            | 枚举简单环     | 近似阶乘级                   | 精确               | 大图不可计算         |
| Bellman-Ford / SPFA | 负权环检测     | (O(N^3)) 或更好              | 判断套利存在较方便 | 不直接优化最大收益   |
| Lap-诱导子图        | 对数残差找热点 | 约 (O(N^2))                  | 推导自然，实验稳定 | 剪枝太深可能漏点     |
| SVD-诱导子图        | 秩一残差找热点 | 完整 SVD 为 (O(N^3))，可优化 | 线性代数解释直观   | 可能受尺度与平滑影响 |

## 十、概率论理论基准推导

大规模图中无法获得全图最优解，因此需要一个理论参照。项目中使用概率论估算随机噪声背景下长度为 (L) 的最佳简单回路收益。

实验数据生成方式为：

[
R_{ij}=\frac{p_j}{p_i}(1+\epsilon_{ij})
]

其中：

[
\epsilon_{ij}\sim U(-\delta,\delta)
]

考虑一条长度为 (L) 的回路：

[
C=i_1\to i_2\to \cdots \to i_L\to i_1
]

其汇率乘积为：

[
\prod_{(i,j)\in C}R_{ij}
========================

\prod_{(i,j)\in C}\frac{p_j}{p_i}(1+\epsilon_{ij})
]

由于回路首尾相接，价格比部分会相互抵消：

[
\prod_{(i,j)\in C}\frac{p_j}{p_i}=1
]

所以回路收益主要由噪声项决定：

[
\prod_{(i,j)\in C}(1+\epsilon_{ij})
]

取对数：

[
\ln \prod_{(i,j)\in C}(1+\epsilon_{ij})
=======================================

\sum_{(i,j)\in C}\ln(1+\epsilon_{ij})
]

当 (\epsilon_{ij}) 较小时：

[
\ln(1+\epsilon_{ij})\approx \epsilon_{ij}
]

因此：

[
\ln P_C \approx \sum_{k=1}^{L}\epsilon_k
]

由于：

[
\epsilon_k\sim U(-\delta,\delta)
]

有：

[
E(\epsilon_k)=0
]

[
Var(\epsilon_k)=\frac{\delta^2}{3}
]

于是：

[
Var\left(\sum_{k=1}^{L}\epsilon_k\right)=\frac{L\delta^2}{3}
]

长度为 (L) 的简单回路数量随 (N) 增长非常快。用极值理论进行近似，可以得到长度为 (L) 的最佳回路收益大致与：

[
\delta L\sqrt{\frac{2}{3}\ln N}
]

同阶。代码中使用的理论估计公式正是：

[
E[P_{\max}]\approx \delta L\sqrt{\frac{2}{3}\ln N}
]

并通过 `calculate_theoretical_expectation` 函数计算理论参考值。

这一部分的价值在于：当全图最优解无法实际计算时，我们仍能用概率论给出“最佳回路大约应达到多少收益”的参照。这样，算法不是孤立地报告一个收益数值，而是可以说它达到了理论最佳收益的较高水平。

## 十一、实验设计

实验分为两部分。

### 11.1 实验 A：小规模绝对精度与耗时实验

实验规模：

[
N=8,10
]

这些规模下可以运行全图 DFS，因此可以得到精确基准。

实验变量包括：

* (N)：节点数量；
* `Trap`：是否注入强套利陷阱；
* `Lap-6Node`：Lap 残差提取 6 个热点节点；
* `Lap-8Node`：Lap 残差提取 8 个热点节点；
* `SVD-8Node`：SVD 秩一残差提取 8 个热点节点；
* 重复次数：5 次；
* 数据每次重新生成。

实验指标包括：

* Base Profit：全图最优收益；
* Base Time：全图 DFS 耗时；
* 剪枝后收益；
* Opt 比例：剪枝收益 / Base 收益；
* 剪枝后 DFS 耗时。

### 11.2 实验 B：大规模硬约束诱导子图实验

实验规模：

[
N=50,100,200
]

这些规模下全图 DFS 不可运行，因此使用：

1. (N=10) 的 DFS 耗时进行阶乘外推，估算全图搜索时间；
2. 概率论公式估算长度 (L=6,8) 的理论最佳收益；
3. 比较 Lap-6Node、Lap-8Node、SVD-8Node 的实际收益与耗时。

实验代码中，大规模实验对 (N=50,100,200) 分别计算 (L=6) 和 (L=8) 的理论收益，并在 `trap=True/False` 两种条件下重复生成数据。

## 十二、实验结果与分析

### 12.1 小规模实验结果

最终实验中，(N=8) 时：

* 无 Trap 情况下，全图收益约为 (2.510%)，耗时约 (17.80) ms；
* Lap-8Node 和 SVD-8Node 都达到 (100.0%) 的收益保留；
* Lap-6Node 仅耗时约 (0.40) ms，但收益保留为 (79.6%)。

有 Trap 情况下：

* 全图收益约为 (13.757%)；
* Lap-8Node 和 SVD-8Node 仍达到 (100.0%)；
* Lap-6Node 达到 (96.3%)，耗时约 (0.41) ms。

这说明在 (N=8) 时，8 个热点节点实际上已经覆盖全图，因此 Lap-8Node 和 SVD-8Node 可以完全保留最优回路；Lap-6Node 虽然极快，但因为节点数更少，存在一定收益损失。

(N=10) 时：

* 无 Trap 情况下，全图 DFS 收益约 (3.257%)，耗时约 (1358.80) ms；
* Lap-6Node 用约 (0.42) ms 达到 (52.6%)；
* Lap-8Node 用约 (17.95) ms 达到 (75.5%)；
* SVD-8Node 用约 (17.84) ms 达到 (72.0%)。

有 Trap 情况下：

* 全图收益约 (14.620%)，耗时约 (1341.69) ms；
* Lap-6Node 达到 (88.9%)；
* Lap-8Node 达到 (93.5%)；
* SVD-8Node 达到 (94.0%)。

这说明，当存在明显套利陷阱时，两种剪枝方法都更容易捕捉到高收益区域；无 Trap 的纯噪声市场中，最佳回路可能更分散，剪枝损失更明显。

### 12.2 大规模实验结果

在大规模实验中，全图 DFS 的估算耗时已经不可接受：

* (N=50)：约 (10^{50}) 年；
* (N=100)：约 (10^{144}) 年；
* (N=200)：接近无限时间。

但剪枝算法仍然保持毫秒级运行。

无 Trap 情况下：

* (N=50)，Lap-8Node 达到 (3.015%)，在三种剪枝结果中为最高，耗时约 (17.49) ms；
* (N=100)，Lap-8Node 达到 (3.060%)，SVD-8Node 达到 (2.910%)，二者接近；
* (N=200)，SVD-8Node 达到 (3.003%)，略高于 Lap-8Node 的 (2.763%)。

有 Trap 情况下：

* (N=50)，Lap-8Node 达到 (14.076%)，SVD-8Node 达到 (13.819%)；
* (N=100)，Lap-8Node 达到 (14.244%)，SVD-8Node 达到 (13.921%)；
* (N=200)，Lap-8Node 达到 (14.515%)，SVD-8Node 达到 (14.002%)。

这说明，在有明显 Trap 的情况下，Lap-8Node 表现非常稳定，通常可以捕捉到接近 (14%) 左右的高收益回路；SVD-8Node 也表现良好，但略低于 Lap-8Node。Lap-6Node 的优势是极快，耗时约 (0.42)–(0.47) ms，但收益通常低于 8Node 版本。

### 12.3 与理论收益的比较

无 Trap 情况下，理论估计的 (L=8) 最佳简单环收益为：

* (N=50)：约 (6.460%)；
* (N=100)：约 (7.009%)；
* (N=200)：约 (7.518%)。

实际剪枝算法在无 Trap 情况下通常达到 (2.7%)–(3.1%) 左右。虽然没有达到理论最优值，但考虑到算法只在 8 个热点节点诱导子图中搜索，而不是全图搜索，这说明它能以极短时间找到收益较高的候选回路。

在 Trap 情况下，理论随机噪声收益不再是主要参照，因为人为注入的结构远强于随机噪声。此时算法能稳定找到约 (13%)–(14.5%) 的收益，说明剪枝方法对强局部套利结构具有较高敏感性。

## 十三、两种剪枝算法的比较

Lap 算法与 SVD 算法各有优势。

Lap 算法的优势是：

* 推导自然；
* 与对数汇率、图中边权、路径收益联系紧密；
* 在 Trap 场景中表现稳定；
* 计算主要为 (O(N^2))，适合大规模预处理。

Lap 算法的不足是：

* 对剪枝深度敏感；
* 极浅剪枝可能漏掉分散型套利；
* 依赖对数残差的局部大小判断。

SVD 算法的优势是：

* 直接利用汇率矩阵的秩一结构；
* 线性代数意义直观；
* 在部分无 Trap 大规模实验中表现接近甚至超过 Lap-8Node；
* 适合作为主成分异常检测方法。

SVD 算法的不足是：

* 完整 SVD 复杂度较高；
* 对尺度敏感，需要行列归一化；
* 可能出现平滑效应，将局部异常分散到全局；
* 某些复杂套利结构中稳定性不如 Lap。

因此，本项目没有证明某一种算法绝对优于另一种，而是提出了一个更合理的结论：

> Lap 与 SVD 是两种不同的线性代数剪枝视角。Lap 更偏图论结构，SVD 更偏矩阵主成分。二者共同证明：用线性代数方法先压缩图，再进行回路搜索，是解决大规模套利路径问题的有效思路。

## 十四、算法适用范围

本方法适用于以下场景：

第一，图较稠密。若大多数货币之间存在直接兑换关系，汇率矩阵信息较完整，Lap 与 SVD 都能较好发挥作用。

第二，节点数较大。若 (N) 很小，直接 DFS 或 Bellman-Ford 即可；剪枝算法的优势主要体现在 (N) 较大时。

第三，目标是快速找到高收益候选回路，而不是严格证明全局最优。

第四，市场存在较明显局部错价。Trap 实验说明，当局部结构较强时，剪枝算法能快速捕捉热点区域。

直接使用 Ford / SPFA 更适合以下场景：

第一，图本身很稀疏。如果边数 (E) 本来就接近 (O(N))，则直接图算法已经很快，没有必要构造完整矩阵。

第二，必须保证不漏报。如果任务是“只要存在任何套利就必须找到”，启发式剪枝存在风险。

第三，数据缺失严重。SVD 与矩阵残差方法依赖矩阵完整性；缺失边过多会影响残差判断。

## 十五、局限性与未来改进

本项目仍有若干局限。

第一，剪枝算法不保证全局最优。热点节点诱导子图保留了局部回路结构，但若真正最优回路分布在残差不够突出的节点中，仍可能被漏掉。

第二，实验数据为模拟数据。虽然 noise 和 trap 参数设计较完整，但仍与真实市场存在差距。真实市场中还存在手续费、滑点、交易容量、订单簿深度等因素。

第三，当前模型默认兑换路径可以立即执行，未考虑时间延迟。

第四，SVD 算法使用完整 `np.linalg.svd`，若要扩展到更大规模，可改用幂迭代或随机 SVD。

第五，概率论理论基准是一种近似。它基于小噪声、独立均匀分布和极值估计，主要用于提供参考，而不是严格的上界证明。

未来可以从以下方向改进：

1. 加入手续费，将套利条件改为：

[
\prod R_{ij}(1-c_{ij})>1
]

2. 使用真实市场汇率数据验证算法；

3. 对剪枝深度 (m=6,8,10,\dots) 做更系统的敏感性分析；

4. 用随机 SVD 或幂迭代降低 SVD 版本复杂度；

5. 引入多轮剪枝：先粗选热点节点，再在其邻域扩展一层；

6. 结合 Bellman-Ford/SPFA，用剪枝图先检测负环，再用 DFS 搜索高收益简单环。

## 十六、结论

本项目围绕“如何在汇率图中快速寻找高收益套利回路”这一问题，构建了一套完整的图论与线性代数结合的方法。

首先，我们将货币兑换市场建模为带权有向图，将套利路径转化为图中的正收益简单回路。其次，我们指出全图搜索在大规模完全图中存在严重计算瓶颈，因此必须进行剪枝。随后，项目提出了两种剪枝算法：Lap 残差剪枝与 SVD 秩一近似剪枝。Lap 算法利用对数汇率中的平均势差结构寻找异常边，SVD 算法利用汇率矩阵的主成分结构寻找异常边。两种算法都不直接保留少数边，而是提取热点节点并构造诱导子图，从而尽量避免破坏套利环结构。

实验方面，项目设置了节点规模、噪声水平、Trap 注入、剪枝深度等多维参数，并进行了重复实验。结果显示，在小规模图中，剪枝算法能显著减少耗时；在大规模图中，当全图 DFS 已经达到天文级估算时间时，Lap-8Node 和 SVD-8Node 仍能在约 18 ms 内找到较高收益回路，Lap-6Node 甚至能在约 0.5 ms 内完成搜索。项目还通过概率论推导估算随机噪声背景下最佳简单回路的理论收益，使大规模实验有了可比较的理论参照。

因此，本项目的主要贡献可以概括为：

> **用线性代数残差定位图中的高潜力区域，用热点节点诱导子图保留回路结构，再用图搜索算法寻找高收益套利回路；并通过多参数实验与概率论理论基准验证该方法在速度和收益之间取得了良好平衡。**

这体现了一个清晰的研究思路：**不是用线性代数取代图论，而是用线性代数帮助图论算法更快地找到值得搜索的地方。**
